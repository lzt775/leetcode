# 围棋比赛安排算法

## 题目描述

这是一道关于围棋比赛安排的算法题。比赛采用单淘汰制，有2^N个参赛选手，进行N轮比赛，最终角逐出一个冠军。

### 比赛要求

1. **观赏性要求**：
   - 前两名（评分最高的两位）选手必须在决赛中对战
   - 前四名选手必须出现在半决赛中
   - 前八名选手必须出现在四分之一决赛中
   - 以此类推，即更高分段的选手比赛应当留到比赛的最后阶段进行

2. **势均力敌要求**：
   - 在满足观赏性要求的前提下，希望尽可能多的比赛对决是"势均力敌"的
   - 一场比赛为"势均力敌"的定义是：两位对战选手的评分差小于或等于某个给定的阈值K

### 输入参数

- `N`：比赛轮数（1 ≤ N ≤ 18）
- `K`："势均力敌"的评分差阈值（1 ≤ K ≤ 10^9）
- `R`：所有选手的评分数组（元素个数为2^N，所有评分值唯一，1 ≤ r ≤ 10^9）

### 输出

返回一个整型，表示在满足所有限制条件的前提下，最多能安排多少场"势均力敌"的比赛。

## 解决方案

### 核心思路

1. **构建比赛树**：按照观赏性要求构建比赛安排，确保前2^i名选手在第i轮出现
2. **统计势均力敌比赛**：遍历每轮比赛，统计评分差≤K的比赛场数

### 算法步骤

1. **选手排序**：按评分降序排序所有选手
2. **构建比赛安排**：
   - 第一轮：按评分顺序安排选手
   - 后续轮次：确保前2^round名选手在第round轮相遇
3. **计算势均力敌比赛**：遍历每轮比赛，检查每对选手的评分差

### 代码结构

- `GoTournamentFinal.java`：主要解决方案类
- `maxCloseMatches()`：主方法，计算最多势均力敌比赛场数
- `buildTournament()`：构建满足观赏性要求的比赛安排
- `countCloseMatches()`：统计势均力敌比赛场数

## 示例

### 示例1
- 输入：`N=2, K=2, R=[9, 1, 6, 4]`
- 输出：`1`
- 说明：只能安排1场势均力敌的比赛（评分差≤2）

### 示例2
- 输入：`N=2, K=5, R=[9, 1, 6, 4]`
- 输出：`3`
- 说明：可以安排3场势均力敌的比赛（评分差≤5）

## 时间复杂度

- **时间复杂度**：O(2^N * N)，其中N是比赛轮数
- **空间复杂度**：O(2^N * N)，用于存储比赛安排表

## 注意事项

1. 算法必须严格满足观赏性要求
2. 在满足观赏性要求的前提下，尽可能多地安排势均力敌的比赛
3. 选手评分必须唯一
4. 比赛轮数N的范围是1到18
